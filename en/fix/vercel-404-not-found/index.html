Vercel: 404: NOT_FOUND

Use this checklist in order. Stop when the error is resolved, and do not apply unrelated system changes from later steps.

Medium urgencyDeployment routing fix

Fast answer

1

Test the generated deployment URL separately from the custom domain.

2

Confirm the project root, framework preset, and emitted route in the build output.

3

Check domain assignment or add the framework-appropriate rewrite for client-only routes.

Confirm the match

Symptoms

  • The deployment succeeds but a page or the root URL returns Vercel 404 NOT_FOUND.
  • Local routing works while a direct production refresh fails.
  • The problem may affect a custom domain, dynamic route, or project root only.

Why it happens

Likely causes

  • The project root or output directory is configured incorrectly.
  • A client-side route lacks the rewrite or server route required for direct requests.
  • The custom domain points to a different Vercel project or deployment.

Diagnostics

Commands to check

npx vercel inspect DEPLOYMENT_URLnpm run buildnpx vercel domains ls

Safety note

Determine whether the Vercel deployment URL works before changing DNS records.
