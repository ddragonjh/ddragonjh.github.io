n8n: webhook not responding

Use this checklist in order. Stop when the error is resolved, and do not apply unrelated system changes from later steps.

High urgencyAutomation outage fix

Fast answer

1

Activate the workflow before using the production URL.

2

Verify WEBHOOK_URL and the reverse proxy route.

3

Send a minimal request with the exact method and path.

Confirm the match

Symptoms

  • The test webhook works but the production webhook fails.
  • The request returns 404, timeout, or no execution appears.
  • A reverse proxy, tunnel, or custom domain is involved.

Why it happens

Likely causes

  • The workflow is not active for production webhooks.
  • The external URL, proxy, or tunnel points to the wrong host.
  • The HTTP method or path does not match the webhook node.

Diagnostics

Commands to check

curl -i https://your-domain/webhook/pathdocker logs n8ndocker compose ps

Safety note

Check the active workflow state and deployment environment variables.
