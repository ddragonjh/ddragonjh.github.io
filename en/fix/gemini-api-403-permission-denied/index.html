Gemini API: 403 PERMISSION_DENIED

Use this checklist in order. Stop when the error is resolved, and do not apply unrelated system changes from later steps.

High urgencyAPI access recovery

Fast answer

1

Confirm which Google Cloud project owns the key and that the Gemini API is enabled.

2

Review application and API restrictions without removing security controls permanently.

3

Create a minimal request from an allowed environment and rotate the key if it was exposed.

Confirm the match

Symptoms

  • Gemini API requests return HTTP 403 with PERMISSION_DENIED.
  • The key exists but fails for one project, region, or referrer.
  • The request may mention a blocked key, restricted access, or missing permission.

Why it happens

Likely causes

  • The API key restriction does not allow the calling app or Gemini API.
  • The Generative Language API is disabled or unavailable for the project.
  • The account, project, region, or usage pattern is restricted.

Diagnostics

Commands to check

gcloud config get-value projectgcloud services list --enabled

Safety note

Never publish a Gemini API key while testing permission settings.
