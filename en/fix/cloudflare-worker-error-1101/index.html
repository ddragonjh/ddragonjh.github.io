Cloudflare Workers: Error 1101: Worker threw exception

Use this checklist in order. Stop when the error is resolved, and do not apply unrelated system changes from later steps.

High urgencyProduction exception fix

Fast answer

1

Stream production logs and reproduce one failing request.

2

Check every required binding and environment variable in the deployed environment.

3

Catch the failing branch, return a controlled error, and deploy again.

Confirm the match

Symptoms

  • The deployed Worker returns an Error 1101 page.
  • Local development works but the production request fails.
  • The problem appears only on a specific route or request payload.

Why it happens

Likely causes

  • An uncaught exception occurs in the request handler.
  • A production binding or environment variable is missing.
  • Node-specific code or an unsupported API reaches the Worker runtime.

Diagnostics

Commands to check

npx wrangler tailnpx wrangler deploy --dry-runnpx wrangler types

Safety note

Use production logs to find the first exception instead of repeatedly redeploying blind changes.
