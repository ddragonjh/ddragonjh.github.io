Docker: could not select device driver with capabilities: [[gpu]]

Use this checklist in order. Stop when the error is resolved, and do not apply unrelated system changes from later steps.

High urgencyContainer GPU setup fix

Fast answer

1

Confirm the host driver sees the GPU with nvidia-smi.

2

Verify the NVIDIA Container Toolkit and Docker runtime configuration.

3

Restart Docker and test the smallest official CUDA container before the application image.

Confirm the match

Symptoms

  • A GPU-enabled container fails before it starts.
  • Docker reports that no device driver satisfies the gpu capability.
  • nvidia-smi may work on the host while Docker cannot expose the GPU.

Why it happens

Likely causes

  • The NVIDIA Container Toolkit is missing or not configured for Docker.
  • Docker was not restarted after the runtime configuration changed.
  • The Compose GPU reservation is missing the required capabilities field.

Diagnostics

Commands to check

nvidia-smidocker infodocker compose config

Safety note

Fix host driver and container-runtime detection before changing application CUDA packages.
