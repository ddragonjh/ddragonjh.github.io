Kubernetes: ImagePullBackOff

Use this checklist in order. Stop when the error is resolved, and do not apply unrelated system changes from later steps.

High urgencyContainer image pull recovery

Fast answer

1

Describe the Pod and read the newest pull event verbatim.

2

Verify the exact image and tag outside the cluster.

3

Repair the namespace pull secret or node-to-registry network path named by the event.

Confirm the match

Symptoms

  • A Pod remains Pending with ImagePullBackOff or ErrImagePull.
  • No application logs appear because the container never starts.
  • Pod events contain the registry's concrete authentication, name, or network error.

Why it happens

Likely causes

  • The image name or tag does not exist in the registry.
  • The namespace lacks a valid image pull secret for a private registry.
  • Cluster nodes cannot resolve or reach the registry.

Diagnostics

Commands to check

kubectl describe pod POD_NAMEkubectl get events --sort-by=.lastTimestampkubectl get secret

Safety note

Changing restart settings cannot fix an image that the node cannot pull.
