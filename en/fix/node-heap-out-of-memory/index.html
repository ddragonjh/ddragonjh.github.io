Node.js: JavaScript heap out of memory

Use this checklist in order. Stop when the error is resolved, and do not apply unrelated system changes from later steps.

High urgencyBuild memory fix

Fast answer

1

Confirm available system or container memory before raising the Node limit.

2

Increase old-space size for one reproducible build.

3

If memory keeps climbing, profile the process and remove the retaining workload.

Confirm the match

Symptoms

  • A build or Node process exits with a fatal heap allocation error.
  • Memory usage grows until the process is terminated.
  • The failure happens on large projects or production builds.

Why it happens

Likely causes

  • The workload exceeds Node's current old-space limit.
  • A plugin, loop, cache, or dependency retains memory unexpectedly.
  • The machine or container has less memory than the process expects.

Diagnostics

Commands to check

node --max-old-space-size=4096 script.jsnode --versionnpm run build

Safety note

A larger heap can unblock a legitimate build, but it does not repair a memory leak.
