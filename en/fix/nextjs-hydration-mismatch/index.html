Next.js: hydration mismatch

Use this checklist in order. Stop when the error is resolved, and do not apply unrelated system changes from later steps.

Medium urgencyReact rendering fix

Fast answer

1

Find the first component named in the hydration warning.

2

Make its initial server and client output deterministic.

3

Move browser-only reads into an effect or load that component without server rendering when justified.

Confirm the match

Symptoms

  • The browser console says server-rendered HTML does not match the client.
  • The page flashes or re-renders immediately after load.
  • The issue appears in production or after a full refresh, but not during client navigation.

Why it happens

Likely causes

  • Rendering depends on the current time, random values, or browser-only state.
  • Invalid HTML nesting is parsed differently by the browser.
  • A client component changes markup before hydration completes.

Diagnostics

Commands to check

npm run buildnpm run startnpm ls react react-dom

Safety note

Do not hide every mismatch with suppressHydrationWarning; fix the nondeterministic output first.
