Claude API: 400 invalid_request_error

Use this checklist in order. Stop when the error is resolved, and do not apply unrelated system changes from later steps.

High urgencyRequest validation fix

Fast answer

1

Log the sanitized error body and identify the first field named in the message.

2

Compare a minimal request against the current Messages API schema.

3

Remove unsupported fields or preserve required thinking blocks exactly before retrying.

Confirm the match

Symptoms

  • Claude API returns HTTP 400 before generating a response.
  • The error points to a field, content block, model, or request format.
  • The same API key works with a minimal example request.

Why it happens

Likely causes

  • The JSON body is malformed or includes a field unsupported by the selected model.
  • An assistant prefill or thinking block was changed in a way the model does not support.
  • The application uses an outdated request shape or invalid content ordering.

Diagnostics

Commands to check

python -m json.tool request.jsonnode -e "JSON.parse(require('fs').readFileSync('request.json'))"

Safety note

Do not retry an unchanged 400 request; correct the client input first.
