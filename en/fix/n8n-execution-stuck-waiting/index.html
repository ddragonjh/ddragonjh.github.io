n8n: execution stuck in running

Use this checklist in order. Stop when the error is resolved, and do not apply unrelated system changes from later steps.

High urgencyWorkflow recovery

Fast answer

1

Open the execution and identify the last node that actually started.

2

Set explicit timeouts on external calls and stop the blocked execution.

3

For queue mode, verify worker, Redis, and database health before retrying.

Confirm the match

Symptoms

  • An execution remains running without reaching the next node.
  • New executions queue behind older work.
  • The UI shows activity but worker logs are quiet or repeatedly reconnecting.

Why it happens

Likely causes

  • A request node is waiting without a practical timeout.
  • Queue-mode workers cannot reach Redis or the database.
  • A code node or sub-workflow never resolves.

Diagnostics

Commands to check

docker logs n8ndocker compose pscurl -I https://target.example

Safety note

Retry only after finding the blocking node, or duplicate executions can compound the outage.
