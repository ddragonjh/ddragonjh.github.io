GitHub: Permission denied (publickey)

Use this checklist in order. Stop when the error is resolved, and do not apply unrelated system changes from later steps.

High urgencyGit authentication fix

Fast answer

1

Test the GitHub SSH connection and note which account responds.

2

Load the correct private key and add its public key to GitHub.

3

Confirm the remote URL and repository permission before retrying the push.

Confirm the match

Symptoms

  • Git clone, pull, or push over SSH fails before repository access is checked.
  • The terminal reports Permission denied (publickey).
  • HTTPS access works but the SSH remote does not.

Why it happens

Likely causes

  • No usable SSH key is loaded into the current agent.
  • The public key is not attached to the signed-in GitHub account.
  • The repository remote points to the wrong host or account.

Diagnostics

Commands to check

ssh -T git@github.comssh-add -lgit remote -v

Safety note

Never upload or share the private half of an SSH key.
