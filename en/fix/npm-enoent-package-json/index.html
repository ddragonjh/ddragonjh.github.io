npm: ENOENT: no such file or directory, open package.json

Use this checklist in order. Stop when the error is resolved, and do not apply unrelated system changes from later steps.

Medium urgencyNode project path fix

Fast answer

1

Print the current directory and list its files.

2

Move to the folder that contains the intended package.json.

3

If the file should exist, restore it from source control rather than creating an empty replacement.

Confirm the match

Symptoms

  • npm exits with ENOENT while trying to open package.json.
  • The command works in the project folder but fails from another directory.
  • A deployment or script starts npm from an unexpected working directory.

Why it happens

Likely causes

  • The current directory does not contain the intended package.json.
  • The project was copied or checked out without required files.
  • A workspace command points to the wrong package path.

Diagnostics

Commands to check

pwddirnpm prefix

Safety note

An empty npm init file does not restore the scripts and dependencies from a missing project manifest.
