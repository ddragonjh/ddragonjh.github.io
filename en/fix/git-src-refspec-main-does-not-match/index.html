Git: src refspec main does not match any

Use this checklist in order. Stop when the error is resolved, and do not apply unrelated system changes from later steps.

Medium urgencyGit push setup fix

Fast answer

1

Check whether the repository has a commit and print the current branch name.

2

Create the intended first commit if the repository is empty.

3

Push the actual branch name or rename it deliberately before setting upstream.

Confirm the match

Symptoms

  • git push names main as a source ref that Git cannot find locally.
  • The repository may have no commits or use a different branch name.
  • The remote exists, but the push stops before remote permission is evaluated.

Why it happens

Likely causes

  • No commit exists yet, so there is no branch ref to push.
  • The current branch is named master or something other than main.
  • The push command contains a misspelled local ref name.

Diagnostics

Commands to check

git statusgit branch --show-currentgit log -1 --oneline

Safety note

Do not create an unrelated branch just to silence the message; confirm the repository's intended branch history.
