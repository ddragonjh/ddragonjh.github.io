Claude API: stream idle timeout - partial response received

Use this checklist in order. Stop when the error is resolved, and do not apply unrelated system changes from later steps.

High urgencyStreaming reliability fix

Fast answer

1

Record the last event and request ID before retrying the incomplete operation.

2

Raise the relevant idle timeout and keep SSE buffering disabled in the proxy.

3

For very long asynchronous work, use a batch workflow instead of one uninterrupted connection.

Confirm the match

Symptoms

  • A streaming Claude response stops after returning only part of the output.
  • The client reports an idle timeout or closed SSE connection.
  • Short requests work while long generations fail behind a proxy or load balancer.

Why it happens

Likely causes

  • A proxy, gateway, or client timeout closes an idle long-lived connection.
  • The application does not consume stream events promptly.
  • A transient network error occurs after the server already returned HTTP 200.

Diagnostics

Commands to check

curl -N https://api.anthropic.com/v1/messagesdate -u

Safety note

A stream can fail after HTTP 200, so handle error events as well as initial status codes.
