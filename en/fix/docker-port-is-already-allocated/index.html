Docker: port is already allocated

Use this checklist in order. Stop when the error is resolved, and do not apply unrelated system changes from later steps.

Medium urgencyLocal development unblock

Fast answer

1

Find the process using the port.

2

Stop the conflicting process or change the host port mapping.

3

Restart the Docker stack after editing compose.

Confirm the match

Symptoms

  • Docker cannot start a container because the host port is busy.
  • The error mentions bind, allocated, or address already in use.
  • Restarting the container does not help.

Why it happens

Likely causes

  • Another container or local app is using the same port.
  • A previous process did not shut down cleanly.
  • The compose file maps multiple services to the same host port.

Diagnostics

Commands to check

docker psdocker compose downlsof -i :3000netstat -ano | findstr :3000

Safety note

Prefer changing the host port over changing the container port.
