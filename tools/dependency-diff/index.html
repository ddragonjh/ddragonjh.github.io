What the ranking means

A version change is a lead, not a verdict

A major version change is highlighted because Semantic Versioning reserves it for incompatible public API changes. Minor and patch updates can still cause regressions, especially when a package does not follow SemVer or when a broad declared range resolves to a different exact version.

Use the output to choose the first controlled test: restore one changed package to the known-good version, rebuild from a clean lockfile, and verify both the original failure signature and the expected result.
