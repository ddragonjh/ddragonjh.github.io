Decision model

Transient does not always mean repeatable

429, 503, and gateway failures can be temporary, but a repeated POST or PATCH may duplicate work if the first request committed before the response was lost. The analyzer separates transient status signals from method safety and checks for an idempotency key.

When Retry-After is present, the server-provided delay becomes the minimum wait. Random jitter is added to candidate schedules so many clients do not return at the same instant. Provider-specific limits and idempotency behavior still take precedence.
